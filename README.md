## 1. Описание проекта

Проект посвящён разработке учебного прототипа нейросетевой системы для распознавания жестового языка по изображениям руки. В качестве исходных данных используется открытый датасет **ASL Alphabet**, содержащий изображения жестов американского жестового алфавита.

В рамках работы реализован полный цикл решения задачи компьютерного зрения: подготовка изображений, формирование обучающей, валидационной и тестовой выборок, создание двух сверточных нейронных сетей, обучение моделей, оценка качества по метрикам и визуализация результатов.

Проект относится к прикладному треку, так как итогом является рабочий программный прототип, который может классифицировать изображение жеста и возвращать предполагаемый класс.

## 2. Цель и задачи

**Цель проекта:** разработать и протестировать прототип нейросетевой модели для распознавания жестов руки по изображениям.

**Задачи проекта:**

1. Подобрать открытый датасет изображений жестов руки.
2. Сформировать рабочую подвыборку данных для обучения и тестирования.
3. Выполнить предобработку изображений: изменение размера, перевод в тензор и нормализацию.
4. Разделить данные на обучающую, валидационную и тестовую выборки.
5. Реализовать две CNN-модели на PyTorch.
6. Обучить модели и сравнить их качество.
7. Рассчитать метрики `accuracy`, `precision`, `recall`, `F1-score`.
8. Построить таблицы, графики и матрицу ошибок.
9. Сохранить лучшую модель и подготовить файл для просмотра архитектуры в Netron.
10. Реализовать функцию предсказания класса жеста по отдельному изображению.

## 3. Используемый датасет

В проекте используется датасет **ASL Alphabet**. Полный датасет достаточно большой, поэтому для учебного прототипа была сформирована ограниченная подвыборка:

| Параметр | Значение |
|---|---|
| Источник данных | ASL Alphabet |
| Тип данных | изображения жестов руки |
| Формат задачи | многоклассовая классификация |
| Используемые классы | A, B, C, D, E, F |
| Количество классов | 6 |
| Максимум изображений на класс | 200 |
| Общий объём подвыборки | 1200 изображений |
| Размер после предобработки | 64×64 пикселя |

Подвыборка используется для того, чтобы быстро проверить работоспособность прототипа без обучения на полном объёме данных.

## 4. Предобработка данных

Для обучения моделей были выполнены следующие шаги:

- отбор изображений по выбранным классам `A-F`;
- приведение изображений к размеру `64×64`;
- перевод изображений в тензоры PyTorch;
- нормализация пикселей;
- разделение данных на обучающую, валидационную и тестовую выборки;
- применение простой аугментации для обучающей выборки.

Разделение данных выполнено в пропорции:

| Выборка | Количество изображений | Доля |
|---|---:|---:|
| Обучающая | 840 | 70% |
| Валидационная | 180 | 15% |
| Тестовая | 180 | 15% |
| Всего | 1200 | 100% |

## 5. Архитектура моделей

Для сравнения были реализованы две сверточные нейронные сети.

### SimpleCNN

Базовая модель с двумя сверточными блоками и классификатором. Она используется как простая базовая линия для сравнения.

### ImprovedCNN

Улучшенная модель с тремя сверточными блоками, `BatchNorm`, `Dropout` и более крупным классификатором. Эта модель лучше извлекает признаки изображений и показывает более высокое качество на тестовой выборке.

Архитектура лучшей модели была дополнительно выведена через `torchinfo.summary()`. Также сохранён файл модели для просмотра графа архитектуры в Netron.

## 6. Результаты обучения и тестирования

Итоговые метрики на тестовой выборке:

| Модель | Accuracy | Precision | Recall | F1-score |
|---|---:|---:|---:|---:|
| SimpleCNN | 0.1889 | 0.1058 | 0.1889 | 0.1299 |
| ImprovedCNN | 0.5333 | 0.5634 | 0.5333 | 0.4328 |

По результатам тестирования лучшей моделью стала **ImprovedCNN**. Она показала более высокие значения всех основных метрик по сравнению с базовой моделью.

## 7. Анализ результатов

Базовая модель `SimpleCNN` показала качество, близкое к случайному угадыванию для шести классов. Это говорит о том, что двух сверточных блоков недостаточно для устойчивого выделения признаков жестов.

Модель `ImprovedCNN` обучилась заметно лучше. У неё снизилась функция потерь, выросла валидационная точность и улучшились итоговые метрики на тестовой выборке. При этом качество всё ещё нельзя считать достаточным для полноценного практического применения: модель путает некоторые жесты, особенно визуально похожие классы.

По матрице ошибок видно, что лучше всего распознаются классы `C` и `F`, а хуже всего - классы `D` и `E`. Это может быть связано с похожим положением пальцев, небольшим числом эпох обучения и ограниченным размером подвыборки.

Для повышения качества в дальнейшем можно:

- увеличить количество эпох обучения;
- использовать больше изображений из полного датасета;
- применить более сильную аугментацию;
- попробовать transfer learning на предобученной модели;
- использовать более сложную архитектуру CNN.

## 10. Как запустить проект

1. Клонировать или скачать репозиторий.
2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Открыть Jupyter Notebook:

```bash
jupyter notebook
```

4. Запустить файл:

```text
5(пром.рез.исслед.)_НФИбд-01-23_Пестова.ipynb
```

5. Выполнить все ячейки сверху вниз.

Датасет должен находиться в папке:

```text
data/asl_dataset/
```

Внутри должны быть подпапки классов `A`, `B`, `C`, `D`, `E`, `F`.
